## Blocking
1. **Cross-owner enqueue after the `playlist_id` re-key.** `jobs.playlist_id references playlists(id)` (single-col) vs `videos`' actual guard `(playlist_id, owner_id) references playlists(id, owner_id)` (`0001:23-32`). `jobs` RLS only checks `owner_id = auth.uid()` (`0008:35-36`), so owner A can enqueue with owner B's `playlist_id` if the UUID leaks; the service worker then runs cross-tenant. Fix: composite FK + `enqueue_job` selects only a playlist owned by `auth.uid()`.
2. **Worker storage seam not safely implementable as specified.** `principal ŌåÉ (ownerId, playlistId)` but cloud storage needs `Principal.indexKey = playlist_key`, not the UUID: `SupabaseBlobStore.objectKey()` = `${p.id}/${p.indexKey}/${key}` (`supabase-blob-store.ts:10-13`); `SupabaseMetadataStore.requirePlaylistId()` resolves `.eq('playlist_key', p.indexKey)` and is not owner-scoped under service_role (`supabase-metadata-store.ts:159-164`). UUID-as-indexKey ŌåÆ wrong path + failed lookup; playlist_key under service_role can select the wrong owner if keys collide. Fix: resolve `playlist_id + owner_id ŌåÆ playlist_key` once, fail if not owned, use playlist-id-aware or owner-scoped metadata.
3. **Summary handler can complete with no indexed video row.** Local reserves via `claimVideoSlot()` then `upsertVideo()` (`pipeline.ts:321-366`). Spec only says cloud writes via `writeArtifact`, but that only calls `updateVideoFields` (`consistency.ts:33-40`) and `merge_video_data` updates 0 rows silently if the row is absent (`0007:88-95`). Blob promoted, job completes, index never shows it. Fix: define who owns `claimVideoSlot` + initial `Video` upsert; make metadata updates assert row count.

## High
1. **Wall-clock cancellation specified against nonexistent signal APIs.** `resolveTranscriptSegments` (`transcript-source.ts:10-14`), `generateSummary`/`generateJson` (SDK timeout only, `gemini.ts:136-146`), `transcribeViaGemini` (`gemini.ts:471-477`), `generateDig` (`dig/generate.ts:170-174`) ŌĆö none take a caller signal. Timeout/lost-lease fires but the paid call continues. Fix: add signal params through transcript/Gemini/dig; compose timeout + lease-lost + shutdown; check before retries/backoff.
2. **Transcript failure taxonomy collapses transient into non-retryable.** ┬¦9 marks "gated + fallback fails" NonRetryable, but `resolveTranscriptSegments` wraps any fallback failure (incl. Gemini 429/network/timeout) into one "transcript unavailable" error (`transcript-source.ts:23-32`) ŌĆö contradicts the same table's "429 retryable" row. Fix: classify caption-fetch vs Gemini-fallback causes; only permanent no-source ŌåÆ NonRetryable.
3. **Cloud dig extraction ignores slide-asset side effects.** `digSection()` calls `resolveSlideTokens()` with `assetsRoot = path.join(outputFolder, 'assets')`, "assets written here" (`dig-section.ts:66-74`). A hosted worker writes cropped slide assets to ephemeral local disk while the promoted dig markdown references now-unavailable assets. Fix: make slide resolution storage-agnostic too ŌĆö return slide bytes/keys or inject an asset writer that stages/promotes to the blob store.

## Medium
1. **`writeArtifact` crash semantics overstated.** ┬¦9 says a stagedŌåÆpromote crash leaves only a temp that `resolveMissing`/repair covers. Actual: metadata is committed *before* promote (`consistency.ts:27-40`), and `resolveMissing()` only classifies missing source blobs as `repair_needed` ŌĆö it does not clean/repair `_staging` temps (`consistency.ts:49-59`). A crash after status `committed` but before promote leaves metadata pointing at a final key with only the temp present, uncovered by read-time repair. Fix: document exact crash states; add a repair path for committed+missing-final+existing-temp, or change ordering/status semantics.
2. **`baseName` ownership underspecified across producer/handler.** Local derives it after `claimVideoSlot()` + local collision check (`pipeline.ts:322-332`); spec has a future (out-of-scope) producer supply it in payload. Handler tests would hand-craft payloads that bypass serial allocation ŌåÆ cloud filenames diverge from local + metadata serials. Fix: move baseName allocation into a defined reservation RPC/contract, tested with the handler.
